## 一、产品概述

求职申请管理看板是一款面向在校大学生的轻量级 Web 应用，帮助用户以可视化看板形式跟踪多条并行求职申请线，清晰掌握每个申请的当前阶段、截止日期和整体进展。

---

## 二、目标用户

| 维度 | 描述 |
|------|------|
| 核心用户 | 正在秋招/春招的应届本科生、研究生 |
| 使用场景 | 同时投递 5–30 家公司，需要跨渠道、跨阶段追踪进度 |
| 设备 | 桌面浏览器为主（Chrome/Edge/Safari） |
| 技术背景 | 普通用户，无需安装任何依赖 |

---

## 三、用户痛点

| # | 痛点 | 当前解法的缺陷 |
|---|------|----------------|
| P1 | 申请数量多，状态难以记忆 | Excel 表格无可视化，扫描成本高 |
| P2 | 截止日期密集，容易遗漏 | 日历应用与申请信息割裂，上下文切换成本高 |
| P3 | 不清楚自己整体"漏斗"情况 | 无法一眼知道面试通过率、Offer 数量 |
| P4 | 同一申请需更新多处记录 | 信息分散在微信收藏、备忘录、Excel |

---

## 四、产品目标

1. **可视化申请漏斗**：用看板呈现从投递到 Offer 的完整流转
2. **截止日期感知**：自动高亮紧迫条目，减少遗漏
3. **数据概览**：一屏内呈现关键求职指标
4. **零学习成本**：打开即用，无需注册，数据本地保存

## 五、功能需求

### 5.1 看板主视图（核心）

**5.1.1 看板列**

| 列名 | 英文 Key | 含义 | 列颜色标识 |
|------|---------|------|-----------|
| 已投递 | applied | 简历已提交，等待反馈 | 蓝灰 |
| 笔试中 | test | 收到笔试邀请/进行中 | 蓝色 |
| 面试中 | interview | 进入面试环节 | 橙色 |
| 已拿 Offer | offer | 获得录用通知 | 绿色 |
| 已拒绝 | rejected | 收到拒绝/主动放弃 | 红灰 |

**5.1.2 卡片信息**

每张申请卡片展示：
- 公司名称（加粗，主标题）
- 岗位名称（副标题）
- 截止日期 + 倒计时徽章（见 5.2）
- 优先级标签（高/中/低，色点标识）
- 投递日期
- 快捷操作：编辑 / 删除

**5.1.3 拖拽操作**

- 支持跨列拖拽：将卡片从一列拖到另一列即更新阶段
- 支持列内排序
- 拖拽时目标列高亮显示
- 拖拽结束后自动持久化到 localStorage

**5.1.4 新建申请**

- 顶部固定"＋ 新增申请"按钮
- 点击弹出模态框，填写字段后保存

### 5.2 截止日期提醒

| 剩余天数 | 视觉表现 |
|---------|---------|
| 已过期 | 红色实心徽章，文字"已过期" |
| ≤ 3 天 | 红色徽章 + 红色边框卡片 + "⚠ N 天" |
| 4–7 天 | 橙色徽章 + "N 天后" |
| > 7 天 | 灰色徽章 + 日期 |
| 无截止日期 | 不显示徽章 |

**顶部紧急提醒栏**：当存在 ≤3 天到期的申请时，在页面顶部显示一条黄色 Banner，列出所有紧急条目。

### 5.3 数据统计面板

位于看板上方，常驻展示 4 个核心指标卡片：

| 指标 | 计算逻辑 |
|------|---------|
| 总投递数 | 全部申请数量 |
| 面试转化率 | (面试中 + Offer) / 总数 × 100% |
| Offer 数量 | stage = offer 的数量 |
| 即将到期 | deadline ≤ 7 天且未拒绝的申请数 |

同时展示各阶段申请数量的横向进度条（漏斗可视化）。

### 5.4 新增/编辑申请模态框

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 公司名称 | 文本 | ✅ | |
| 申请岗位 | 文本 | ✅ | |
| 当前阶段 | 下拉选择 | ✅ | 五个阶段 |
| 截止日期 | 日期选择器 | ❌ | 笔试/面试截止时间 |
| 投递日期 | 日期选择器 | ✅ | 默认今天 |
| 优先级 | 单选（高/中/低） | ✅ | 默认中 |
| 招聘链接 | URL | ❌ | 原职位页面链接 |
| 备注 | 多行文本 | ❌ | 面试官、地点、注意事项等 |

**交互：**
- 编辑时回填现有数据
- 删除时弹出二次确认

### 5.5 数据持久化

- 所有数据存储在 `localStorage`（key: `job-board-applications`）
- 应用初始化时若 localStorage 为空，加载 10 条预置示例数据
- 每次操作（增删改拖拽）后自动保存


## 八、技术选型

| 层次 | 技术 | 理由 |
|------|------|------|
| 框架 | Vue 3 (Composition API) | 响应式、组件化，生态成熟 |
| 构建 | Vite | 快速冷启动，HMR 即时热更新 |
| 状态 | Pinia | Vue 3 官方推荐，比 Vuex 更简洁 |
| 拖拽 | vuedraggable@next (v4) | Vue 3 适配，底层 Sortable.js |
| 持久化 | localStorage | 无后端，零配置 |
| 样式 | 原生 CSS（CSS Variables）| 零依赖，设计 Token 统一管理 |
| 图标 | 内联 SVG / Unicode Emoji | 零额外请求 |
