# 求职申请管理看板

这是我为求职季做的一款申请进度管理工具。它把不同公司、不同岗位、不同阶段的申请信息放到一个看板里，帮助我更清楚地知道自己投了哪些岗位、哪些快到截止时间、哪些需要继续跟进。

在线体验：[https://job-application-board-six.vercel.app](https://job-application-board-six.vercel.app)

## 项目介绍

在求职过程中，我发现最容易出问题的不是“不知道投什么”，而是信息太分散：岗位链接在招聘平台里，沟通记录在聊天软件里，截止日期在邮件或表格里，面试安排又可能在日历里。投递数量一多，就很容易漏掉重要节点。

所以我做了这个求职申请管理看板，希望它像一个轻量的个人求职工作台：

- 集中记录公司、岗位、城市、截止日期和备注
- 用看板方式管理申请阶段，让进度一眼可见
- 标记即将截止的申请，减少遗漏风险
- 通过统计概览了解当前求职整体状态
- 支持账号登录，不同用户的数据相互隔离

这个项目更偏向真实使用场景，而不是单纯展示页面效果。我希望它能帮助用户在高压的求职周期里，把琐碎的信息整理清楚，把注意力放回准备简历、笔试和面试本身。

## 使用方式

打开在线链接后，可以直接注册一个账号开始使用。

<img width="2555" height="1398" alt="520a6091a9be5b7379621174f7a34a43" src="https://github.com/user-attachments/assets/1f92beb2-93fa-48d8-a850-cc573334fb38" />

首次进入系统后，可以新增自己的求职申请，填写公司、岗位、城市、申请阶段、截止日期、岗位链接和备注等信息。申请会以卡片形式出现在看板中，可以按阶段查看，也可以通过**拖拽**调整进度。

<img width="2556" height="1394" alt="1f4ad93fd0a8b22df02c424e61a1f19e" src="https://github.com/user-attachments/assets/856223cf-c5e2-449f-805d-7db20e8b2fd4" />

当某些申请距离截止日期较近时，页面会给出提醒，帮助我优先处理更紧急的事项。顶部的统计信息可以快速看到当前总申请数、面试进展和不同阶段的分布情况。

## 适合场景

- 校招、实习、春招、秋招申请管理
- 同时投递多个公司和岗位时记录进度
- 面试前整理岗位信息和备注
- 查看自己当前求职漏斗，判断下一步重点

## 本地运行

如果需要在本地运行项目，可以按下面的方式启动：

```bash
npm install
npm run dev
```

后端和数据库需要额外配置环境变量，示例见 `.env.example`。线上体验版本已经完成部署，可以优先使用在线链接查看效果。

## 关于项目

这是一个从需求整理、界面设计、数据建模、前后端开发到线上部署都由我独立完成的项目。相比堆砌功能，我更关注它是否真的能解决一个具体问题：让求职申请不再散落在各个地方，而是变成一个清晰、可跟进、可复盘的流程。
